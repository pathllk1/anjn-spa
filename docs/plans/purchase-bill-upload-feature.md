## 1. Feature Overview
After a purchase bill is successfully saved or updated, the system will provide an optional file upload feature on the success modal. This allows users to attach a scanned copy of the physical bill (PDF or JPEG) directly to the digital record.

### Key Constraints:
*   **Trigger:** Post-save/update success modal in the Purchase system.
*   **File Format:** `.pdf`, `.jpg`, `.jpeg` only.
*   **Max Size:** 200 KB.
*   **Storage Path:** `uploads/[firmid]/[userid]/[filename]` (local) + Optional Cloud Mirror (Backblaze).
*   **Error Handling:** Validation for size, type, and upload failures.

As requested, we will replicate the robust Backblaze B2 upload logic found in the database controller.

### The Backblaze Upload Function (Reference):
```javascript
async function uploadToBackblazeB2(buffer, fileName) {
  const keyId    = String(process.env.B2_APPLICATION_KEY_ID || '').trim();
  const appKey   = String(process.env.B2_APPLICATION_KEY    || '').trim();
  const bucketId = String(process.env.B2_BUCKET_ID          || '').trim();
  const prefix   = String(process.env.B2_BUCKET_PREFIX      || '').trim().replace(/^\/+|\/+$/g, '');

  if (!keyId || !appKey) throw new Error('B2_APPLICATION_KEY_ID / B2_APPLICATION_KEY not configured');
  if (!bucketId)         throw new Error('B2_BUCKET_ID is not configured');

  // Step 1: Authorise account
  let auth;
  const authResp = await axios.get('https://api.backblazeb2.com/b2api/v2/b2_authorize_account', {
    headers: { Authorization: `Basic ${Buffer.from(`${keyId}:${appKey}`).toString('base64')}` }
  });
  auth = authResp.data;

  // Step 2: Get upload URL
  const urlResp = await axios.post(`${auth.apiUrl}/b2api/v2/b2_get_upload_url`, { bucketId }, {
    headers: { Authorization: auth.authorizationToken }
  });
  const uploadInfo = urlResp.data;

  // Step 3: Upload the file
  const targetName = prefix ? `${prefix}/${fileName}` : fileName;
  const sha1 = createHash('sha1').update(buffer).digest('hex');

  const uploadResp = await axios.post(uploadInfo.uploadUrl, buffer, {
    headers: {
      Authorization:       uploadInfo.authorizationToken,
      'X-Bz-File-Name':    encodeURIComponent(targetName),
      'Content-Type':      'application/octet-stream', // Adjusted for generic file
      'Content-Length':    buffer.length,
      'X-Bz-Content-Sha1': sha1,
    }
  });
  return `${auth.downloadUrl}/file/${uploadResp.data.bucketName}/${encodeURIComponent(targetName)}`;
}
```

## 4. Detailed Implementation Steps

### Step 1: Multer Middleware Configuration
In the purchase routes, configure multer to restrict:
*   `limits: { fileSize: 200 * 1024 }` (200 KB).
*   `fileFilter`: only `image/jpeg` and `application/pdf`.

### Step 2: Backend Controller Logic
1.  Verify the Bill belongs to the User's firm.
2.  Generate path: `uploads/${firmId}/${userId}/${Date.now()}-${file.originalname}`.
3.  Ensure directories exist using `fs.mkdirSync({ recursive: true })`.
4.  Write file locally.
5.  (Optional) Trigger Backblaze upload.
6.  Update `Bill` record in MongoDB with the local path or cloud URL.

### Step 3: Frontend Modal Update
1.  Add `<input type="file" id="bill-file-input" accept=".pdf,.jpg,.jpeg" class="...">`.
2.  Add an "Upload Scanned Bill" button that is disabled until a file is selected.

4.  On success, show a "File Uploaded" checkmark.
